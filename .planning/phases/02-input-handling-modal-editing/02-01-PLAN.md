<task type="auto">
  <name>Task 2: Implement mode-aware input routing</name>
  <files>src/input.c</files>
  <read_first>src/input.c</read_first>
  <action>
Modify handle_input() function to route keys based on app->app_mode:

1. In Normal mode (app->app_mode == MODE_NORMAL):
   - Route 'i' and KEY_ENTER to enter_insert_mode()
   - Route navigation keys (h/j/k/l, arrows, Tab, etc.) normally
   - Pass through other keys to existing navigation handlers

2. In Insert mode:
   - Route Escape (27) to exit_insert_mode()
   - Arrow keys for text cursor navigation
   - All printable characters for text input

Add helper functions:
```c
void enter_insert_mode(Application* app) {
    app->app_mode = MODE_INSERT;
    // Update status bar
}

void exit_insert_mode(Application* app) {
    app->app_mode = MODE_NORMAL;
    // Update status bar
}
```

Update handle_input() switch statement to check mode first, then route keys accordingly.
  </action>
  <verify>
    <automated>grep -E "enter_insert_mode|exit_insert_mode|app->app_mode" src/input.c</automated>
  </verify>
  <done>Pressing 'i' or Enter enters Insert mode, pressing Esc returns to Normal mode, mode state stored in app->app_mode</done>
</task>
