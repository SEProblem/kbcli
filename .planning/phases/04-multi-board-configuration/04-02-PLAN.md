<task type="auto">
  <name>Task 1: Create config module with JSON parsing</name>
  <files>src/config.c, src/config.h</files>
  <action>
Create new config module:

1. Define Config struct in config.h:
```c
typedef struct {
    char board_directory[MAX_PATH];
    int auto_save;
    char keybindings[64];  // JSON string of keybindings
    char default_board[MAX_NAME];
} Config;
```

2. Add default keybindings in config.h (used if config file missing):
```c
#define DEFAULT_KEYBINDINGS "{\"up\":\"k\",\"down\":\"j\",\"left\":\"h\",\"right\":\"l\",\"create\":\"n\",\"delete\":\"d\"}"
```

3. In config.c implement:
   - `get_config_path(char* buffer, size_t size)` - Returns ~/.config/kanban-cli/config
   - `config_load(Config* config)` - Loads JSON config, applies defaults if missing/corrupt
   - `config_save(const Config* config)` - Saves config to JSON (atomic write)
   - `config_get_board_directory(const Config* config)` - Returns board directory (config or default)

4. Parse JSON using basic string parsing (no external library needed) - find key/value pairs between { }

5. Use atomic write pattern from storage.c for config_save()

Example config file format:
```json
{
  "board_directory": "/custom/path/boards",
  "auto_save": true,
  "keybindings": {"up":"k","down":"j","left":"h","right":"l"},
  "default_board": "work"
}
```
  </action>

</verify>
  <done>Config module loads/saves JSON configuration</done>
</task>
