    int column_index;  // 0=To Do, 1=In Progress, 2=Done

} Selection;

**handle_input(Board *board, int key):**
- 'o' (lowercase): Call task_create below current (D-05)
- 'O' (uppercase, Shift+o): Call task_create above current (D-06)
- 'd' or 'x': Call task_delete for selected task (D-08, agent discretion)
- Arrow keys: Move selection up/down/left/right
- 'q': Return quit signal to event loop
- Tab/Shift+Tab: Move between columns

**read_task_title(void):**
- Use ncurses input field: call wgetnstr() on input pad
- Display prompt: "Task title: "
- Allow up to 255 chars (per Task.title size)
- Return to caller for insertion

**Integration with storage:**
- After task_create or task_delete: call board_save(board, board->filename)
- Auto-save on every change per D-14

**Flow:**
1. User presses 'o' or 'O'
2. Input mode activates for title entry
3. User types title, presses Enter
4. Task created in correct position
5. board_save() called immediately (auto-save per STO-03)
6. Renderer updates board display

Update src/main.c to wire everything together:

**Add global state:**
```c
static Board global_board;
static Selection current_selection = {0, 0};
```

**Event loop pattern:**
```c
int running = 1;
while (running) {
    render_board(&global_board);
    int key = getch();
    
    switch (key) {
        case 'o':
        case 'O':
        case 'd':
        case 'x':
        case KEY_DOWN:
        case KEY_UP:
        case KEY_LEFT:
        case KEY_RIGHT:
        case '\t':
        case KEY_BTAB:
            handle_input(&global_board, key, &current_selection);
            break;
        case 'q':
        case 'Q':
            running = 0;
            break;
    }
}
```

**Key behaviors:**
- Every create/delete/move triggers board_save() inside handle_input
- render_board called each loop iteration for visual sync
- 'q' or 'Q' exits loop cleanly, calls endwin() via atexit
