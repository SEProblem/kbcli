- Update task_count (unchanged)
- Used for 'j'/'k' movement with modification or dedicated reorder keys

**move_up(Board *board, Selection *sel):**
- User presses 'j' (down) or 'k' (up) with modifier or dedicated key
- Simpler: Arrow Up/Down navigates selection, 'J' (Shift+j) moves task up within column
- Swap positions with adjacent task in same column

**move_down(Board *board, Selection *sel):**
- 'K' (Shift+k) moves task down within column
- Swap positions with adjacent task in same column

**Key mapping (per D-11 "Tasks can be reordered within a column via movement keys"):**
- 'h'/'l' = between columns (inter-column movement)
- 'j'/'k' = navigate (no data change)
- 'J'/'K' = reorder within column (intra-column movement)
- This mirrors vim conventions where Shift+letter does "opposite" or "bigger" action

After reordering: board_save() called to persist.

Final integration in src/main.c:

**Startup sequence:**
1. Get default board path: `~/.config/kanban-cli/boards/default.md`
2. Ensure directory exists: `mkdir -p ~/.config/kanban-cli/boards/`
3. Call board_load(&global_board, filepath)
4. If file doesn't exist: board_init() creates empty 3-column board
5. board_save() called immediately to create file

**Movement key integration (update handle_input):**
- 'h': Call move_left(&global_board, &current_selection)
- 'l': Call move_right(&global_board, &current_selection)  
- 'J': Call move_up(&global_board, &current_selection)
- 'K': Call move_down(&global_board, &current_selection)
- After each: board_save(&global_board, global_board.filename)

**Quit handling (APP-02, APP-03):**
- 'q' triggers running=0
- atexit(cleanup) calls endwin()
- Signal handlers (SIGINT/SIGTERM) call endwin() then exit
- Terminal fully restored to original state
